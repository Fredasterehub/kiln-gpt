# Reporting Contract

Stage 5 turns run artifacts into a grounded delivery handoff.

## Reporter Role

The reporter assembles:

- what was actually delivered
- what was actually validated
- what remains risky or incomplete
- what the operator needs next

It does not invent polish by hiding uncertainty.

## Required Inputs

- `docs/VISION.md`
- `docs/master-plan.md`
- execution archives
- validation report and test results
- any missing-credential or correction artifacts

## Required Outputs

- `reports/final/final-report.md`
- optional supporting deployment or handoff notes under `reports/final/`

## Final Report Sections

- scope delivered
- validation summary
- deployment or runtime posture
- known risks and limitations
- concise operator handoff notes

## Reporting Rules

- validation claims must map to real evidence
- limitations must stay visible
- if hosted deployment did not happen, say so plainly
- elegance is allowed; inflation is not
